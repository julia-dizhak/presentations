JSX

  • JSX is a JS syntax extension that looks similar to XML
  • XML has the benefit of balanced opening and closing tags. This helps make large trees easier to read than function calls or object literals;
  • JSX it’s a compromise that allows us to write our React components in a syntax similar to HTML, which is a pretty good deal.

JSX can be used

To render HTML tags

                         
                            const element = <div className="wrap"/>
                            ReactDOM.render(element, document.getElementById('mountNode'));

To render React Component

                         
                            const element = <CustomComponent className="wrap"/>
                            ReactDOM.render(element, document.getElementById('mountNode'));

A couple of things about JSX

  1. in HTML, you use class, and in JSX, you use className
  2. Expressions {}.
  3. You don't have to use JSX with React. You can just use plain JS. However, we recommend using JSX because it is a concise and familiar syntax for defining tree structures with attributes.

Difference between state and props

Props

  1. are immutable
  2. come to the component from parent component/components
  3. are initialization data

State

  1. is mutable
  2. should or can't be accessed from child components

Should a Component have state?

state is optional, state increases complexity and reduces predictability.

A Component without state is preferable, avoid having many stateful Components

    Component types
  1. stateless — only props, no state. All logic revolves around received props. This makes them very easy to follow and test.
  2. stateful — both props and state. Also called state managers. They are in charge of client-server communication, processing data and responding to user events.

Data flow

  1. React implements a one-way data flow. This means that data is passed from the top-down, through props, from the top component to its children, so on and so forth;
  2. Instead of React components talking to each other, React components cooperate through the parent;

Router

  1. React.js itself doesn’t provide any routing utilities, but there’re several free, open-sources libs that works perfectly well
  2. We will discuss react-router which has been developed by React.js team: https://github.com/reactjs/react-router
  3. React Router keeps your UI in sync with the URL. It has a simple API with powerful features like lazy code loading, dynamic route matching, and location transition handling built right in

Router path

  1. A route path is a string pattern that is used to match a URL (or part of one)
  2. Route paths are interpreted literally, except next special symbols:
    1. :paramName – matches a URL segment up to the next /, ?, or #. The matched string is called a param
    2. () – wraps a portion of the URL that is optional
    3. * – matches all characters (non-greedy) up to the next character in the pattern, or to the end of the URL if there is none, and creates a splat param
    4. ** - matches all characters (greedy) until the next /, ?, or # and creates a splat param
